# 本地化

ScavLib 的本地化层把你的物品名、键位标签以及任何其他 UI 字符串翻译成玩家的语言。你其实已经在
不知不觉中用过它了:`CustomItemBuilder.DisplayName` / `Description` 和 `KeyBindBuilder.DisplayName`
/ `Description` 的字典重载会直接喂给它。本页直接讲这条路径,外加给翻译者用的 JSON 文件方式,
以及生成翻译模板的辅助工具。

提供翻译有两种方式:代码里的**内联字典**,和磁盘上的 **JSON 语言文件**。两者共存——少量标签用
内联字符串,想让翻译者在源码之外工作时用 JSON 文件。

## 内联字符串

一条翻译就是一个从**语言代码**到文本的字典。构建器直接接收它:

```csharp
var names = new Dictionary<string, string>
{
    { "EN", "Old Flask" },
    { "ZH", "旧酒壶" },
};

CustomItemBuilder.Canteen("mymod_flask", "MyMod")
    .DisplayName(names)
    .Register();
```

对于不是物品或键位标签的字符串,直接以一个 key 注册它们,并通过游戏的 locale 查表读回:

```csharp
using ScavLib.i18n;

LocaleManager.RegisterString("mymod_greeting", new Dictionary<string, string>
{
    { "EN", "Hello!" },
    { "ZH", "你好!" },
});

// 在别处:
string text = Locale.GetOther("mymod_greeting");
```

底层上,物品显示名走 `LocaleManager.RegisterItem`、键位标签走 `RegisterString`——你很少自己调
它们,但若有需要它们是公开的。

!!! tip "永远提供一个 `EN` 条目"

    当玩家的语言不在你的字典里时,ScavLib 按顺序回退:当前语言代码 → `EN` → `English` →
    第一个条目。一个 `EN` 键能为你没翻译的每种语言保证一个合理的回退。

## JSON 语言文件

对较大的 Mod,把翻译作为 JSON 发布,这样翻译者永远不用碰代码。在你 Mod 内的 `lang/` 文件夹里
每种语言放一个文件,然后注册该文件夹:

```text
MyMod/
  lang/
    EN.json
    ZH.json
```

```csharp
LocaleManager.AutoRegister("MyMod", modFolderPath);
```

`AutoRegister` 总会加载 `EN.json`,然后加载匹配当前游戏语言的文件(`zh-CN` 的游戏区域设置也会
尝试 `zh`)。每个文件是一个 JSON 对象,为每类游戏文本设有保留段,外加一个自由的 `other` 段:

```json
{
  "items": {
    "mymod_flask": "Old Flask"
  },
  "other": {
    "mymod_greeting": "Hello!"
  }
}
```

可识别的顶层保留段有 `items`、`buildings`、`moodles`、`other`、`character`、`pauseQuotes`。
你放在**根部**(这些段之外)的任何键都被当作 `other` 条目。嵌套对象按点号扁平化,所以
`{"ui": {"title": "..."}}` 会变成键 `ui.title`。

## 翻译何时生效

ScavLib 会在游戏加载某种语言之后,把你注册的一切注入游戏的 locale,所以在你 Mod 的 `OnEnabled`
里注册是正确的——在首次语言加载前注册的内容会在加载时应用,之后注册的内容会被立即刷入。

!!! note "物品名在切换语言时实时更新"

    当玩家在游戏内切换语言时,ScavLib 会就地把每个已注册自定义物品的显示名重新同步到新语言。
    所以自定义物品在切换语言后会立刻显示正确的名称,无需重载。

## 生成翻译模板

与其手写 `items` 段,你可以导出一份包含每个已注册物品 id 的模板再填文本。`ExportTemplate`
写出一个带 `TODO` 占位的 JSON 骨架——而且若文件已存在,它会**合并**,保留已完成的翻译、只补上
缺失的键:

```csharp
using System.IO;
using ScavLib.i18n;

TranslationTemplateGenerator.ExportTemplate(Path.Combine(modFolder, "lang", "EN.json"));
```

注册物品后运行一次以搭起文件骨架,然后把 JSON 交给翻译者;之后再次运行会接纳新物品而不覆盖
他们的工作。

## 运行时刷新 UI

如果你在游戏运行时改了字符串、并希望屏幕文本立即更新,`LocaleManager.RefreshAllUI()` 会把当前
翻译重新应用到场景中每个本地化 UI 元素。你通常不需要它——切换语言时的实时重同步已覆盖常见情形
——但它为动态文本而存在。

## 下一步去哪

本地化支撑着你的[物品](items.md)、[键位](input.md)与[命令](commands.md)上的标签。最后一个要
讲的系统是[存档数据](save.md)——把你 Mod 的状态与玩家存档一同持久化。

*[LocaleManager]: ScavLib 的本地化枢纽——注册字符串并把它们注入游戏 locale。
*[TranslationTemplateGenerator]: 导出每个已注册物品 id 的 JSON 翻译模板的辅助工具。
*[Locale]: 游戏原生本地化对象,ScavLib 把翻译注入其中。
*[RegisterItem]: 注册一个物品本地化名称与描述的 LocaleManager 调用。
*[RegisterString]: 以一个 key 注册任意本地化字符串的 LocaleManager 调用。
*[AutoRegister]: 为当前语言加载某 Mod 的 lang/*.json 文件的 LocaleManager 调用。
